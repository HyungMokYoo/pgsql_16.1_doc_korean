변환 생성

변환 생성 - 새 인코딩 변환을 정의합니다.

개요

CREATE [ DEFAULT ] CONVERSION nameFOR source_encoding TO dest_encoding FROM function_name

설명

CREATE CONVERSION은 두 문자 집합 인코딩 간의 새 변환을 정의합니다.

DEFAULT로 표시된 변환은 클라이언트와 서버 간의 자동 인코딩 변환에 사용할 수 있습니다. 이 사용법을 지원하려면 인코딩 A에서 B로 ,인코딩 B에서 A로 두 가지 변환을 정의해야 합니다.

변환을 만들 수 있으려면 함수에 대한 EXECUTE 권한과 대상 스키마에 대한 CREATE 권한이 있어야 합니다.

파라미터

DEFAULT

DEFAULT 절은 이 변환이 이 특정 소스에서 대상으로의 인코딩에 대한 기본값임을 나타냅니다. 인코딩 쌍에 대한 스키마에는 기본 인코딩이 하나만 있어야 합니다.

name

변환의 이름입니다. 변환 이름은 스키마로 한정할 수 있습니다. 그렇지 않은 경우 변환은 현재 스키마에 정의됩니다. 변환 이름은 스키마 내에서 고유해야 합니다.

source_encoding

소스 인코딩 이름입니다.

destination_encoding

대상 인코딩 이름입니다.

function_name

변환을 수행하는 데 사용되는 함수입니다. 함수 이름은 스키마로 한정할 수 있습니다. 그렇지 않은 경우 경로에서 함수를 조회합니다.

함수는 다음과 같은 서명을 가져야 합니다:

conv_proc( 정수, -- 소스 인코딩 ID 정수, -- 대상 인코딩 ID cstring, -- 소스 문자열(널로 끝난 C 문자열) 내부, -- 대상(널로 끝난 C 문자열로 채움) 정수, -- 소스 문자열 길이 부울 -- 참이면 변환에 실패해도 오류를 던지지 않음 ) 함수는 정수를 반환합니다;

반환 값은 성공적으로 변환된 소스 바이트 수입니다. 마지막 인수가 거짓이면 함수는 유효하지 않은 입력에 대해 오류를 반환해야 하며 반환값은 항상 소스 문자열 길이와 같습니다.

참고

소스 인코딩이나 대상 인코딩 모두 SQL_ASCII일 수 없으며, SQL_ASCII '인코딩 '과 관련된 경우에 대한 서버의 동작은 하드 와이어되어 있습니다.

사용자 정의 변환을 제거하려면 DROP CONVERSION을 사용합니다.

변환을 만드는 데 필요한 권한은 향후 릴리스에서 변경될 수 있습니다.

예제

myfunc를 사용하여 UTF8 인코딩에서 LATIN1로 변환을 만들려면 다음과 같이 하세요:

CREATE CONVERSION myconv FOR 'UTF8' TO 'LATIN1' FROM myfunc;

호환성

CREATE CONVERSION은 PostgreSQL 확장입니다. SQL 표준에는 목적과 구문이 매우 유사한 CREATE CONVERSION문이 없지만 CREATE TRANSLATION문이 있습니다.
