20.15. 사전 설정 옵션 #

다음 "매개변수 "는 읽기 전용입니다. 따라서 샘플 postgresql.conf 파일에서 제외되었습니다. 이러한 옵션은 특정 애플리케이션, 특히 관리 프런트엔드에서 관심을 가질 수 있는 PostgreSQL 동작의 다양한 측면을 보고합니다. 대부분은 PostgreSQL이컴파일될 때 또는 설치될 때 결정됩니다.

block_size (정수) #.

디스크 블록의 크기를 보고합니다. 서버를 빌드할 때 BLCKSZ 값에 의해 결정됩니다. 기본값은 8192바이트입니다. 공유_버퍼와 같은 일부 구성 변수의 의미는 block_size의 영향을 받습니다. 자세한 내용은 섹션 20.4를 참조하세요.

data_checksums (boolean) # #

이 클러스터에 대해 데이터 체크섬이 활성화되었는지 여부를 보고합니다. 자세한 내용은 데이터 체크섬을 참조하세요.

data_directory_mode (정수) #.

유닉스 시스템에서 이 매개변수는 서버 시작 시 데이터 디렉터리( data_directory로 정의)의 권한을 보고합니다. (Microsoft Windows에서는 항상 0700이 표시됩니다.) 자세한 내용은 그룹 액세스를 참조하세요.

debug_assertions (boolean) #.

어설션이 활성화된 상태로 PostgreSQL이 빌드되었는지 여부를 보고합니다. PostgreSQL이 빌드될 때 매크로 USE_ASSERT_CHECKING이 정의된 경우(예: config 옵션 --enable-cassert로 수행)에 해당됩니다. 기본적으로 PostgreSQL은 어설션 없이 빌드됩니다.

integer_datetimes (boolean) #.

64비트 정수 날짜 및 시간을 지원하여 PostgreSQL이 빌드되었는지 여부를 보고합니다. PostgreSQL 10부터 이 옵션은 항상 켜져 있습니다.

in_hot_standby (boolean) # #

서버가 현재 핫 스탠바이 모드에 있는지 여부를 보고합니다. 이 옵션이 켜져 있으면 모든 트랜잭션이 강제로 읽기 전용이 됩니다. 세션 내에서 서버가 기본 서버로 승격된 경우에만 변경할 수 있습니다. 자세한 내용은 섹션 27.4를 참조하세요.

max_function_args (정수) # #

함수 인수의 최대 개수를 보고합니다. 서버를 빌드할 때 FUNC_MAX_ARGS 값에 따라 결정됩니다. 기본값은 100개 인자입니다.

max_identifier_length (정수) # #

최대 식별자 길이를 보고합니다. 서버 빌드 시 NAMEDATALEN 값보다 작은 값으로 결정됩니다. 기본값은 64이므로 기본 max_identifier_length는 63바이트이며, 멀티바이트 인코딩을 사용하는 경우 63자보다 작을 수 있습니다.

max_index_keys (정수) # #

인덱스 키의 최대 개수를 보고합니다. 서버를 구축할 때 INDEX_MAX_KEYS 값에 따라 결정됩니다. 기본값은 32개 키입니다.

segment_size (정수) # #.

파일 세그먼트 내에 저장할 수 있는 블록(페이지) 수를 보고합니다. 서버를 빌드할 때 RELSEG_SIZE값에 따라 결정됩니다. 세그먼트 파일의 최대 크기(바이트)는 segment_size에 block_size를 곱한 값이며, 기본값은 1GB입니다.

server_encoding (문자열) #

데이터베이스 인코딩(문자 집합)을 보고합니다. 데이터베이스가 생성될 때 결정됩니다. 일반적으로 클라이언트는 client_encoding 값만 신경 쓰면 됩니다.

server_version (문자열) #

서버의 버전 번호를 보고합니다. 서버를 빌드할 때 PG_VERSION 값에 따라 결정됩니다.

서버_버전_넘버 (정수) #

서버의 버전 번호를 정수로 반환합니다. 서버 빌드 시 PG_VERSION_NUM의 값에 의해 결정됩니다.

shared_memory_size (정수) # #

기본 공유 메모리 영역의 크기를 가장 가까운 메가바이트 단위로 반올림하여 보고합니다.

shared_memory_size_in_huge_pages (정수) #.

기본 공유 메모리 영역에 필요한 거대한 페이지 수를 지정된 huge_page_size에 따라 보고합니다. 거대한 페이지가 지원되지 않는 경우 -1이 됩니다.

이 설정은 Linux에서만 지원됩니다. 다른 플랫폼에서는 항상 -1로 설정됩니다. Linux에서 대용량 페이지 사용에 대한 자세한 내용은 섹션 19.4.5를 참조하세요.

ssl_library (문자열) # #.

이 인스턴스에서 SSL이 현재 구성되지 않았거나 사용 중이 아니더라도 이 PostgreSQL 서버가 구축된 SSL 라이브러리의 이름(예: OpenSSL)을 보고하거나, 없는 경우 빈 문자열을 보고합니다.

wal_block_size (정수) #.

WAL 디스크 블록의 크기를 보고합니다. 서버를 빌드할 때 XLOG_BLCKSZ의 값에 의해 결정됩니다. 기본값은 8192바이트입니다.

wal_segment_size (정수) #.

미리 쓰기 로그 세그먼트의 크기를 보고합니다. 기본값은 16MB입니다. 자세한 내용은 섹션 30.5를 참조하세요.
