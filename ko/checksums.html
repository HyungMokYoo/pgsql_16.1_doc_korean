30.2. 데이터 체크섬 #

30.2.1. 체크섬의 오프라인 활성화

기본적으로 데이터 페이지는 체크섬으로 보호되지 않지만 클러스터에 대해 선택적으로 이 기능을 활성화할 수 있습니다. 이 기능을 활성화하면 각 데이터 페이지에는 페이지가 기록될 때 업데이트되고 페이지를 읽을 때마다 확인되는 체크섬이 포함됩니다. 데이터 페이지만 체크섬으로 보호되며 내부 데이터 구조와 임시 파일은 보호되지 않습니다.

체크섬은 일반적으로 initdb를 사용하여 클러스터를 초기화할 때 활성화됩니다. 나중에 오프라인 작업으로 활성화 또는 비활성화할 수도 있습니다. 데이터 체크섬은 전체 클러스터 수준에서 활성화 또는 비활성화되며 데이터베이스나 테이블에 대해 개별적으로 지정할 수 없습니다.

클러스터의 현재 체크섬 상태는 SHOW data_checksums 명령을 실행하여 읽기 전용 구성 변수 data_checksums의 값을 확인하여 확인할 수 있습니다.

페이지 손상으로부터 복구를 시도할 때 체크섬 보호를 우회해야 할 수도 있습니다. 이렇게 하려면 일시적으로 구성 매개변수 ignore_checksum_failure를 설정합니다.

30.2.1. 오프라인에서 체크섬 활성화 #.

오프라인 클러스터에서 데이터 체크섬을 활성화 또는 비활성화하고 체크섬을 확인하는 데 pg_checksums애플리케이션을 사용할 수 있습니다.
