pg_isready

pg_isready - PostgreSQL 서버의 연결 상태 확인

시놉시스

pg_isready[연결-옵션...][옵션...]

설명

pg_isready는 PostgreSQL 데이터베이스 서버의 연결 상태를 확인하는 유틸리티입니다. 종료 상태는 연결 확인 결과를 지정합니다.

옵션

-d dbname
--dbname=dbname

연결할 데이터베이스의 이름을 지정합니다. dbname은 연결 문자열일 수 있습니다. 이 경우 연결 문자열 매개 변수가 충돌하는 명령줄 옵션을 재정의합니다.

-h 호스트 이름
--host=호스트 이름

서버가 실행 중인 컴퓨터의 호스트 이름을 지정합니다. 값이 슬래시로 시작하면 이 값이 Unix 도메인 소켓의 디렉토리로 사용됩니다.

-p port
--port=port

서버가 연결을 수신 대기 중인 TCP 포트 또는 로컬 Unix 도메인 소켓 파일 확장명을 지정합니다. 기본값은 PGPORT환경 변수의 값을 사용하거나 설정하지 않은 경우 컴파일 시 지정한 포트(일반적으로 5432)를 사용합니다.

-q
--quiet

상태 메시지를 표시하지 않습니다. 스크립팅할 때 유용합니다.

-t seconds
--timeout=초

연결을 시도할 때 서버가 응답하지 않는다고 반환하기 전에 대기할 최대 시간(초)입니다. 0으로 설정하면 비활성화됩니다. 기본값은 3초입니다.

-U 사용자 이름
--사용자 이름=사용자 이름

기본값 대신 사용자 사용자 이름으로 데이터베이스에 연결합니다.

-V
--버전

pg_isready 버전을 출력하고 종료합니다.

-?
--help

pg_isready 명령줄 인자에 대한 도움말을 표시하고 종료합니다.

종료 상태

서버가 정상적으로 연결을 수락하면 0, 서버가 연결을 거부하면(예: 시작 중) 1, 연결 시도에 대한 응답이 없으면 2, 연결 시도가 실패하면(예: 잘못된 매개 변수로 인해) 3을 셸에 반환합니다.

환경

대부분의 다른 PostgreSQL유틸리티와 마찬가지로 pg_isready도 libpq에서지원하는 환경 변수를 사용합니다( 34.15절 참조).

환경 변수 PG_COLOR는 진단 메시지에서 색상을 사용할지 여부를 지정합니다. 가능한 값은 항상, 자동 및 절대로입니다.

참고

서버 상태를 확인하기 위해 정확한 사용자 이름, 비밀번호 또는 데이터베이스 이름 값을 제공할 필요는 없지만 잘못된 값을 제공하면 서버는 연결 시도 실패를 기록합니다.

예시

표준 사용법:

$ pg_isready/tmp:5432 - 연결 수락$ echo $?0

시작 시 연결 매개 변수를 사용하여 PostgreSQL 클러스터로 실행합니다:

$ pg_isready -h localhost -p 5433localhost:5433 - 연결 거부$ echo $?1

응답하지 않는 PostgreSQL 클러스터에 대한 연결 매개 변수를 사용하여 실행:

pg_isready -h someremotehostsomeremotehost:5432 - 응답 없음$ echo $?2
