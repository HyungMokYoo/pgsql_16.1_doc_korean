구독 변경
이전 UpSQL 명령홈 다음

ALTER SUBSCRIPTION

ALTER SUBSCRIPTION - 구독의 정의를 변경합니다.

개요

ALTER SUBSCRIPTION name CONNECTION'conninfo' ALTER SUBSCRIPTION name SET PUBLICATION publication_name [, ...] [ WITH ( publication_option [= value] [, ... ] ) ] ALTER SUBSCRIPTION name ADD PUBLICATION publication_name [, ....] [ WITH ( publication_option [= value] [, ... ] ) ] ALTER SUBSCRIPTION name DROP PUBLICATION publication_name [, ...] [ WITH ( publication_option [= value] [, ... ] ) ] ALTER SUBSCRIPTION name DROP PUBLICATION publication_name [, ...] [, .... ) ] ] ALTER SUBSCRIPTION name REFRESH PUBLICATION [ WITH ( refresh_option [= value] [, ... ] ) ] ALTER SUBSCRIPTION name ENABLE ALTER SUBSCRIPTION name DISABLE ALTER SUBSCRIPTION name SET ( subscription_parameter [= value] [, .... ] ) ALTER SUBSCRIPTION name SKIP ( skip_option = value ) ALTER SUBSCRIPTION name OWNER TO { new_owner | CURRENT_ROLE | CURRENT_USER | SESSION_USER } ALTER SUBSCRIPTION name RENAME TO new_name

설명

ALTER SUBSCRIPTION은 CREATE SUBSCRIPTION에서 지정할 수 있는 대부분의 구독 속성을 변경할 수 있습니다.

ALTER SUBSCRIPTION을 사용하려면 구독을 소유해야 합니다. 구독의 이름을 변경하거나 소유자를 변경하려면 데이터베이스에 대한 CREATE 권한이 있어야 합니다. 또한 소유자를 변경하려면 새 소유자 역할로 SET ROLE을 설정할 수 있어야 합니다. 구독에 password_required=false가 있는 경우 수퍼유저만 구독을 수정할 수 있습니다.

발행물을 새로 고칠 때 더 이상 발행물의 일부가 아닌 관계는 제거되며 테이블 동기화 슬롯이 있는 경우 제거됩니다. 원격 호스트에서 구독에 할당된 리소스를 해제하려면 이러한 슬롯을 제거해야 합니다. 네트워크 고장이나 기타 오류로 인해 PostgreSQL이슬롯을 제거할 수 없는 경우 오류가 보고됩니다. 이 상황에서 계속 진행하려면 사용자는 작업을 다시 시도하거나 구독에서 슬롯을 분리하고 구독 삭제에 설명된 대로 구독을 삭제해야 합니다.

명령어 구독 변경 ... 새로 고침 및 구독 변경 ... {SET|ADD|DROP} 새로 고침 옵션이 true인 PUBLICATION ...명령은 트랜잭션 블록 내에서 실행할 수 없습니다. 이 명령은 구독에 2_phase커밋이 활성화된 경우에도 copy_data가 false가 아닌 한 실행할 수 없습니다. 실제 2단계 상태를 확인하려면 pg_subscription의 subtwophasestate열을 참조하세요.

파라미터

name

속성을 변경할 구독의 이름입니다.

연결 'conninfo'

이 절은 원래 구독 생성에 의해 설정된 연결 문자열을 대체합니다. 자세한 내용은 여기를 참조하세요.

SET PUBLICATION 발행물_이름
ADD PUBLICATION 발행물_이름
DROP PUBLICATION 발행물_이름

이 양식을 사용하면 구독 중인 발행물 목록이 변경됩니다. SET은전체 발행물 목록을 새 목록으로 바꾸고, ADD는 발행물 목록에 발행물을 추가하며, DROP은 발행물 목록에서 발행물을 제거합니다. 사용자가 나중에 추가할 수 있도록 ADD 및 SET 변형에 존재하지 않는 발행물을 지정할 수 있도록 허용합니다. 자세한 내용은 구독 만들기를참조하세요. 기본적으로 이 명령은 REFRESH PUBLICATION처럼 작동합니다.

publication_option은 이 작업에 대한 추가 옵션을 지정합니다. 지원되는 옵션은 다음과 같습니다:

새로 고침 (부울)

false이면 명령이 테이블 정보 새로 고침을 시도하지 않습니다. 그러면 REFRESH PUBLICATION을 별도로 실행해야 합니다. 기본값은 true입니다.

또한 암시적 새로 고침 작업을 제어하기 위해 REFRESH PUBLICATION에 설명된 옵션을 지정할 수 있습니다.

게시 새로 고침

게시자로부터 누락된 테이블 정보를 가져옵니다. 이렇게 하면 구독 생성 또는 REFRESH PUBLICATION의 마지막 호출 이후 구독 대상 발행물에 추가된 테이블의 복제가 시작됩니다.

refresh_option은 새로 고침 작업에 대한 추가 옵션을 지정합니다. 지원되는 옵션은 다음과 같습니다:

COPY_DATA (부울)

복제가 시작될 때 구독 중인 발행물의 기존 데이터를 복사할지 여부를 지정합니다. 기본값은 true입니다.

이전에 구독한 테이블은 테이블의 행 필터 WHERE 절이 수정된 경우에도 복사되지 않습니다.

copy_data = true가 origin매개변수와 상호 작용하는 방식에 대한 자세한 내용은 참고 사항을 참조하세요.

기존 데이터를 바이너리 형식으로 복사하는 방법에 대한 자세한 내용은 구독 생성의 바이너리매개변수를 참조하세요.

ENABLE

이전에 비활성화된 구독을 활성화하여 트랜잭션이 끝날 때 논리적 복제 워커를 시작합니다.

DISABLE

실행 중인 구독을 비활성화하여 트랜잭션이 끝날 때 논리적 복제 작업자를 중지합니다.

SET ( 구독_파라미터 [ =값] [, ... ] )

이 절은 원래 구독을 만들 때 설정한 매개변수를 변경합니다. 자세한 내용은 여기를 참조하세요. 변경할 수 있는 파라미터는 slot_name, synchronous_commit, binary, streaming, disable_on_error, password_required, run_as_owner 및 origin입니다. 수퍼유저만 password_required = false로 설정할 수 있습니다.

SKIP ( skip_option = 값 )

원격 트랜잭션의 모든 변경 사항 적용을 건너뜁니다. 들어오는 데이터가 제약 조건을 위반하는 경우, 문제가 해결될 때까지 논리적 복제가 중지됩니다. ALTER SUBSCRIPTION ... SKIP 명령을 사용하면 논리 복제 작업자는 트랜잭션 내의 모든 데이터 수정 변경을 건너뜁니다. 이 옵션은 구독자에서 2_phase를활성화하여 이미 준비된 트랜잭션에는 영향을 미치지 않습니다. 논리적 복제 작업자가 트랜잭션을 성공적으로 건너뛰거나 트랜잭션을 완료하면 LSN( pg_subscription.subskiplsn에 저장됨)이 지워집니다. 논리적 복제 충돌에 대한 자세한 내용은 섹션 31.5를 참조하십시오.

skip_option은 이 작업에 대한 옵션을 지정합니다. 지원되는 옵션은 다음과 같습니다:

LSN (PG_LSN)

논리적 복제 작업자가 변경을 건너뛸 원격 트랜잭션의 완료 LSN을 지정합니다. 완료 LSN은 트랜잭션이 커밋되거나 준비되는 LSN입니다. 개별 하위 트랜잭션 건너뛰기는 지원되지 않습니다. NONE으로 설정하면 LSN이 초기화됩니다.

new_owner

구독의 새 소유자의 사용자 이름입니다.

new_name

구독의 새 이름입니다.

부울 유형의 매개변수를 지정할 때 = 값부분을 생략할 수 있으며, 이는 TRUE를 지정하는 것과 동일합니다.

예제

구독으로 구독하는 발행물을 insert_only로 변경합니다:

ALTER SUBSCRIPTION mysub SET PUBLICATION insert_only;

구독을 비활성화(중지)합니다:

ALTER SUBSCRIPTION mysub DISABLE;

호환성

ALTER SUBSCRIPTION은 PostgreSQL확장입니다.
